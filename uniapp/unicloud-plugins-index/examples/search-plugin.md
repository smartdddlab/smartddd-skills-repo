# 搜索插件示例

本示例展示如何使用 unicloud-plugins-index 技能搜索和选择插件。

## 示例 1: 查找 UI 组件库

### 用户需求

```
用户: 我需要为我的 uni-app 项目选择一个 UI 组件库,有什么推荐吗?
```

### 助手响应

```
助手: 根据你的需求,我推荐以下 UI 组件库:

1. **uView UI** (推荐用于 Vue2 项目)
   - 下载量: 1,375,751+
   - 评分: 4.9/5.0
   - 特点: 最流行的组件库,60+ 精美组件,完美支持 nvue
   - 文档: https://uviewui.com/

2. **uView Plus** (推荐用于 Vue3 项目)
   - 下载量: 275,487+
   - 评分: 4.8/5.0
   - 特点: 支持 Vue3、鸿蒙 Next、uni-app x,70+ 组件
   - 文档: https://uview-plus.jiangruyi.com/

3. **FirstUI** (推荐用于高性能需求)
   - 下载量: 576,562+
   - 评分: 4.9/5.0
   - 特点: 超高性能、超高颜值,包含组件库+功能库+模板库+物料市场
   - 文档: https://www.firstui.cn/

你的项目使用的是 Vue2 还是 Vue3?这样我可以给出更具体的建议。
```

## 示例 2: 查找支付插件

### 用户需求

```
用户: 我需要在 uni-app 中集成支付功能,应该使用什么插件?
```

### 助手响应

```
助手: 对于 uni-app 支付功能,推荐使用 **uni-pay**:

**uni-pay 统一支付**

**功能特点:**
- 统一支付 API,无需研究各平台支付
- 支持微信支付、支付宝支付、Apple Pay
- 多端统一接口(App、小程序)
- 内置于 uni-admin 支付统计报表

**安装方法:**
```bash
# 从 HBuilderX 插件市场导入
# 或使用 npm
npm install uni-pay
```

**快速开始:**
```js
// 在云函数中引用
const uniPay = require('uni-pay')

// 发起支付
const result = await uniPay.pay({
  provider: 'wxpay',
  outTradeNo: 'ORDER_123',
  totalFee: 100,
  body: '商品描述'
})
```

**官方文档:**
https://uniapp.dcloud.net.cn/uniCloud/unipay

你需要集成哪个支付平台?我可以提供更详细的配置指导。
```

## 示例 3: 查找身份认证插件

### 用户需求

```
用户: 我需要实现用户登录注册功能,应该用什么插件?
```

### 助手响应

```
助手: 对于用户登录注册功能,推荐使用 uni-id 系列插件:

## uni-id 系列插件

### 1. 核心组件

**uni-id-common** (核心模块,服务空间自动内置)
- token 管理(生成、校验、刷新)
- 权限管理(RBAC 模型)
- 密码加密

**uni-id-co** (云对象)
- 多种登录方式(账户密码、手机验证码、三方登录)
- 用户注册和管理
- 密码管理
- 实名认证

**uni-id-pages** (页面模板)
- 完整的登录、注册、修改密码页面
- 个人中心页面
- 实名认证页面
- 适配 PC 宽屏和移动端

### 2. 辅助插件

**uni-config-center** (配置中心)
- 统一配置管理

**uni-captcha** (验证码)
- 图形验证码
- 短信验证码
- 邮箱验证码

### 3. 推荐组合

**完整方案:**
```
uni-id-common + uni-id-co + uni-id-pages + uni-config-center + uni-captcha
```

**快速开始:**
1. uni-id-common 自动内置,无需安装
2. 从插件市场导入 uni-id-co 和 uni-id-pages
3. 配置 uni-config-center
4. 在 pages.json 中配置 uni-id-pages 路由

**官方文档:**
https://doc.dcloud.net.cn/uniCloud/uni-id/summary.html

你希望实现哪些登录方式?我可以提供具体的配置指导。
```

## 示例 4: 查找后台管理插件

### 用户需求

```
用户: 我需要为我的 uniCloud 项目添加后台管理功能
```

### 助手响应

```
助手: 推荐使用 **uni-admin** 后台管理框架:

## uni-admin

**简介:**
基于 uni-app 和 uniCloud 的应用后台管理开源框架,专为 uniCloud 开发者设计。

**主要功能:**
- 开箱即用的后台管理模板
- 用户、角色、权限管理
- 注册用户统计报表
- 丰富的插件扩展生态
- 基于 uni-id 的角色权限系统
- 支持多端管理后台

**安装方法:**
1. 从 HBuilderX 插件市场导入 uni-admin
2. 配置 uniCloud 服务空间
3. 初始化数据库
4. 访问管理后台

**依赖插件:**
- uni-id-common
- uni-id-co

**官方文档:**
https://doc.dcloud.net.cn/uniCloud/admin

**插件市场:**
https://ext.dcloud.net.cn/plugin?id=3268

**扩展插件:**
- uni-admin-users - 用户管理
- uni-admin-roles - 角色管理
- uni-admin-permissions - 权限管理

你的项目需要哪些后台管理功能?我可以帮你规划插件组合。
```

## 示例 5: 对比插件

### 用户需求

```
用户: uView UI 和 FirstUI 有什么区别?我应该选哪个?
```

### 助手响应

```
助手: uView UI 和 FirstUI 的主要区别:

## 对比表格

| 特性 | uView UI | FirstUI |
|------|---------|---------|
| **Vue 版本** | Vue2 | Vue2 / Vue3 |
| **组件数量** | 60+ | 60+ (含功能库) |
| **下载量** | 1,375,751+ | 576,562+ |
| **评分** | 4.9 | 4.9 |
| **nvue 支持** | 完美支持 | 完美支持 |
| **特点** | 最流行,社区活跃 | 超高性能,物料市场 |
| **额外功能** | 工具函数 | 功能库+模板库+物料市场 |

## 详细对比

### uView UI
**优点:**
- 下载量最高,社区最活跃
- 文档完善,示例丰富
- 学习资源多
- 社区问题容易找到解决方案

**适合场景:**
- Vue2 项目
- 需要稳定的组件库
- 团队协作项目(学习成本低)

### FirstUI
**优点:**
- 性能极致优化
- 功能最全(组件+功能+模板)
- 物料市场提供更多资源
- 支持 Vue2 和 Vue3

**适合场景:**
- 对性能要求高的项目
- 需要丰富功能的项目
- Vue3 项目(也可以考虑 uView Plus)

## 推荐建议

**选择 uView UI 如果:**
- 你的项目使用 Vue2
- 你希望有最多的社区支持
- 你需要一个稳定可靠的组件库

**选择 FirstUI 如果:**
- 你对性能有极致要求
- 你需要更多功能(如模板、物料)
- 你的项目可能升级到 Vue3

你的项目具体情况如何?我可以给出更针对性的建议。
```
