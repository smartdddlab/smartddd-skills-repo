# 数据抓取方法论

本文档说明如何从 DCloud 插件市场和相关来源抓取插件数据。

## 数据来源

### 1. DCloud 官方文档

**优先级:** ⭐⭐⭐⭐⭐

**适用插件:**
- uni-id 系列
- uni-admin
- uni-pay
- uni-sms
- uni-email
- uni-push

**抓取方法:**
```
1. 访问官方文档页面
2. 提取插件的功能描述、API 文档
3. 整理为结构化数据
```

**示例:**
- uni-id 官方文档: https://doc.dcloud.net.cn/uniCloud/uni-id/summary.html
- uni-admin 官方文档: https://doc.dcloud.net.cn/uniCloud/admin
- uni-pay 官方文档: https://uniapp.dcloud.net.cn/uniCloud/unipay

### 2. DCloud 插件市场

**优先级:** ⭐⭐⭐⭐

**适用插件:**
- UI 组件库
- 通用组件
- 项目模板
- 行业解决方案

**抓取方法:**
```
1. 访问插件详情页
2. 提取插件名称、描述、下载量、评分
3. 提取作者信息
4. 提取平台支持信息
```

**示例:**
- uView UI: https://ext.dcloud.net.cn/plugin?id=1593
- uni-admin: https://ext.dcloud.net.cn/plugin?id=3268

**限制:**
- 插件市场页面需要浏览器渲染
- 无法直接用 webReader 获取详细信息
- 建议使用 agent-browser 或手动记录

### 3. 插件官网

**优先级:** ⭐⭐⭐⭐

**适用插件:**
- UI 组件库(通常有独立官网)
- 开源项目

**抓取方法:**
```
1. 从插件市场或文档找到官网链接
2. 访问官网首页
3. 提取产品介绍、功能特性
4. 提取文档链接
```

**示例:**
- uView UI: https://uviewui.com/
- uView Plus: https://uview-plus.jiangruyi.com/
- ThorUI: https://thorui.cn/
- FirstUI: https://www.firstui.cn/

**优点:**
- 信息最全面
- 更新及时
- 有详细的功能说明

### 4. Web 搜索

**优先级:** ⭐⭐⭐

**适用场景:**
- 补充插件信息
- 查找官方文档链接
- 获取最新版本信息

**搜索关键词:**
```
"{插件名} DCloud 官方文档"
"{插件名} uniCloud 教程"
"{插件名} API 文档"
```

## 抓取工具

### 1. webReader (推荐)

**适用场景:**
- 访问静态网页
- 获取网页文本内容
- 读取官方文档

**使用方法:**
```
1. 使用 mcp__web_reader__webReader 工具
2. 传入 URL 参数
3. 获取网页内容(支持 markdown 和 text 格式)
```

**优点:**
- 无需浏览器环境
- 速度快
- 支持 markdown 格式

**缺点:**
- 无法执行 JavaScript
- 无法获取动态内容
- 部分网站可能无法访问

### 2. agent-browser

**适用场景:**
- 需要 JavaScript 渲染的页面
- 需要交互操作
- 抓取动态内容

**使用方法:**
```
1. 确保已安装 Chrome 或 Chromium
2. 使用 agent-browser skill
3. 执行浏览器操作
```

**优点:**
- 支持完整页面渲染
- 可以模拟用户操作
- 可以截图

**缺点:**
- 需要浏览器环境
- 速度较慢
- 资源消耗大

### 3. WebSearch

**适用场景:**
- 查找官方文档链接
- 搜索特定插件信息

**使用方法:**
```
1. 使用 WebSearch 工具
2. 传入搜索关键词
3. 从结果中筛选官方来源
```

**优点:**
- 快速找到相关资源
- 可以发现新的数据源

**缺点:**
- 结果需要人工筛选
- 可能包含过时信息

## 数据抓取流程

### 第一阶段: 规划

**步骤:**
1. 确定要抓取的插件列表
2. 为每个插件确定优先级(P0-P3)
3. 确定数据来源和抓取工具
4. 设计数据结构

**输出:**
- 插件清单列表(如:plugin-list.md)
- 数据字段定义

### 第二阶段: 执行

**步骤:**
1. 按优先级顺序抓取
2. 使用 webReader 访问官方文档
3. 使用 WebSearch 查找补充信息
4. 访问插件官网获取详细信息
5. 整理为结构化数据

**输出:**
- 原始数据文件
- 抓取日志

### 第三阶段: 整理

**步骤:**
1. 统一数据格式
2. 补充缺失字段
3. 验证数据准确性
4. 更新 JSON 索引文件

**输出:**
- plugins-index.json(最终索引文件)

### 第四阶段: 维护

**步骤:**
1. 定期检查插件更新
2. 更新过时数据
3. 添加新插件
4. 记录变更日志

**输出:**
- 更新后的索引文件
- 变更记录

## 数据字段

### 必填字段

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| `id` | string | 插件唯一标识 | "uni-id-common" |
| `name` | string | 插件名称 | "uni-id-common" |
| `displayName` | string | 显示名称 | "uni-id 公共模块" |
| `category` | string | 分类 | "unicloud" |
| `author` | string | 作者 | "DCloud前端团队" |
| `description` | string | 描述 | "uni-id 账户体系的核心..." |
| `features` | array | 功能列表 | ["token 管理", "权限管理"] |
| `platforms` | array | 支持平台 | ["app", "h5", "mp-weixin"] |
| `documentation` | string | 文档链接 | "https://doc.dcloud..." |
| `marketplace` | string | 市场链接 | "https://ext.dcloud..." |
| `lastUpdated` | string | 最后更新 | "2026-03-21" |

### 可选字段

| 字段 | 类型 | 说明 | 示例 |
|------|------|------|------|
| `version` | string | 版本 | "4.0.0+" |
| `downloads` | number | 下载量 | 11496343 |
| `stars` | number | 评分 | 4.8 |
| `tags` | array | 标签 | ["auth", "security"] |
| `installCommand` | string | 安装命令 | "npm install..." |
| `quickStart` | string | 快速开始 | "// 在云函数中..." |
| `dependencies` | array | 依赖插件 | ["uni-config-center"] |
| `compatibility` | object | 兼容性 | {"vue": "2.x / 3.x"} |
| `dataTables` | array | 数据表 | ["uni-id-users"] |
| `officialSite` | string | 官网 | "https://uviewui.com/" |
| `pages` | array | 页面列表 | ["login/login"] |
| `configPath` | string | 配置路径 | "uniCloud/..." |

## 数据质量标准

### 准确性

- [ ] 下载量、评分数据来源可靠
- [ ] 文档链接可访问
- [ ] 描述与插件实际功能一致
- [ ] 版本信息准确

### 完整性

- [ ] 所有必填字段都已填写
- [ ] 功能列表完整
- [ ] 平台支持信息完整
- [ ] 依赖关系明确

### 时效性

- [ ] 最后更新时间在 3 个月内
- [ ] 下载量数据相对较新
- [ ] 文档链接指向最新版本

### 一致性

- [ ] 所有插件使用相同的数据结构
- [ ] 命名风格统一
- [ ] 分类标准一致

## 抓取优先级

### P0 (最高优先级)

**插件类型:**
- uni-id 系列
- 主流 UI 组件库

**数据来源:**
- 官方文档
- 插件官网

**完成标准:**
- 所有必填字段完整
- features 列表详细
- quickStart 代码可用

### P1 (高优先级)

**插件类型:**
- uniCloud 核心插件
- 数据可视化组件

**数据来源:**
- 官方文档
- Web 搜索

**完成标准:**
- 基本信息完整
- 文档链接准确

### P2 (中优先级)

**插件类型:**
- UTS 插件
- 通用组件

**数据来源:**
- 插件市场
- 官方文档(如有)

**完成标准:**
- 基本信息
- 下载量和评分

### P3 (低优先级)

**插件类型:**
- 项目模板
- 行业解决方案

**数据来源:**
- 插件市场
- Web 搜索

**完成标准:**
- 基本信息
- 简介

## 错误处理

### 网络访问失败

**原因:**
- 网络连接问题
- 网站暂时不可用
- 被防火墙拦截

**解决方案:**
1. 重试访问
2. 使用代理
3. 尝试其他数据源
4. 手动记录信息

### 数据不一致

**原因:**
- 不同来源数据冲突
- 数据过时
- 版本差异

**解决方案:**
1. 以官方文档为准
2. 以最新来源为准
3. 标注数据来源和日期
4. 记录不一致情况

### 信息缺失

**原因:**
- 插件文档不完整
- 插件市场信息缺失
- 官网无法访问

**解决方案:**
1. 查看插件源码
2. 联系插件作者
3. 使用默认值或标注"未知"
4. 降低该插件的优先级

## 维护策略

### 定期更新

**频率:**
- 核心插件:每月
- 常用插件:每季度
- 其他插件:每半年

**内容:**
- 更新下载量和评分
- 检查文档链接有效性
- 更新版本信息
- 添加新插件

### 版本控制

**策略:**
- 每次更新增加 PATCH 版本号
- 添加新插件增加 MINOR 版本号
- 数据结构变更增加 MAJOR 版本号

**示例:**
```json
{
  "version": "1.2.0",
  "lastUpdated": "2026-03-21T17:40:00Z",
  "totalPlugins": 16
}
```

### 变更日志

**记录内容:**
- 新增的插件
- 删除的插件
- 更新的字段
- 数据来源变更

**示例:**
```markdown
## v1.2.0 (2026-03-21)
- 新增 5 个 uniCloud 核心插件
- 更新 UI 组件库下载量数据
- 修复 uni-admin 文档链接
```
